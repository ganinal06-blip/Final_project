## Итоги работы над проектом

В ходе выполнения проекта каждая участница команды внесла вклад в разработку бота-модератора. Ниже приведены итоговые результаты работы каждого участника.

### Лиза (тимлид)

По итогу проекта Лиза:

* Разработала пользовательскую логику бота и взаимодействие с администратором
* Реализовала команды:

  * /start — приветственное сообщение и инструкция по использованию бота
  * загрузку `.txt`-файла со списком разрешённых пользователей
  * автоматическую реакцию бота на вход новых участников в группу
  * /clean — команду ручной очистки группы от неразрешённых пользователей
* Реализовала обработку входящих файлов и сообщений (`handlers.py`)
* Написала утилиты для парсинга списка разрешённых пользователей (`utils.py`)
* Настроила структуру проекта и точку входа приложения (`main.py`, `bot_app.py`)
* Подготовила конфигурационные файлы проекта (`.env`, `requirements.txt`)
* Реализовала Docker-конфигурацию (`Dockerfile`, `docker-compose.yml`)
* Выполняла координацию работы команды и финальное тестирование бота

---

### Маша

По итогу проекта Маша:

* Реализовала основную логику модерации пользователей
* Разработала сервис модерации (`services.py`), включающий:

  * фильтрацию пользователей по белому списку
  * подготовку списка пользователей на удаление
  * бан/удаление пользователей из группы
* Реализовала работу с базой данных:

  * настройку подключения (`db.py`)
  * описание моделей данных (`models.py`)
  * репозитории для работы с пользователями и логами (`repository.py`)
* Настроила логирование действий бота (кто и за что был удалён)
* Написала unit-тесты для проверки логики:

  * работы сервисов
  * моделей
  * репозиториев
* Проверила корректность работы команды /clean и автоматической модерации

---

### Совместная работа

В ходе проекта мы совместно:

* Обсуждали и принимали решения по логике модерации
* Проводили код-ревью
* Выполняли финальное тестирование бота в реальном Telegram-чате
* Проверяли соответствие проекта требованиям задания
* Оформили документацию проекта (README, схемы, описание логики)
